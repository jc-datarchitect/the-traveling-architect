# Contributing to The Traveling Architect

Thank you for your interest in improving The Traveling Architect! This project is a space where architectural research meets geospatial data, and I welcome contributions to make it more accurate, efficient, and global.

## How to Contribute

Whether you are an architect, a data enthusiast, or a traveler, there are several ways to help:

### 1. Reporting Issues
If you find a bug, a missing landmark, or an inaccuracy in the data, please [open an issue](https://github.com/JC-datarchitect/the-traveling-architect/issues). When reporting:
*   Describe the issue clearly.
*   If it's a data error, please provide the correct coordinates or context.

### 2. Adding Landmarks
The core of this project is its dataset (`tta_destinations.json`). To add new architectural landmarks:
*   Fork the repository.
*   Add the new entry following the existing JSON structure.
*   Submit a Pull Request (PR) with a brief description of the landmark's significance.

### 3. Improving the Logic
If you have ideas for optimizing the route planner or adding new analytical features:
*   Feel free to suggest changes via issues.
*   If you plan a significant code change, please open an issue first to discuss the implementation.

## Code of Conduct
Please ensure all contributions remain professional and constructive. This is a space for sharing knowledge and improving architectural documentation.
